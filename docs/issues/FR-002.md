# [FR-002] Cinematic Camera System

## Labels
`MUS`, `polish`, `camera`

## User Story
As a player, I want the camera to organically breathe and violently shake so that the game feels visceral and "alive" rather than static.

## Proposed Solution

### Overview
Implement a multi-layered camera transformation system applied at the beginning of the `drawGame()` loop. It should act like an After Effects Null Object parent.

### Implementation Flow
1. Create `cameraBreathe` object utilizing figure-8 math (`sin(t)`, `sin(2t)`) and noise functions for organic drift.
2. Create `cameraShake` for combat impacts (short bursts of high-frequency decay).
3. Create `surgeShake` for extreme environmental chaos.
4. Apply translations and rotations in `drawGame()` before drawing the world.

### Technical Approach
```javascript
ctx.save();
// 1. Organic Base Drift
ctx.translate(center.x + cameraBreathe.offsetX, center.y + cameraBreathe.offsetY);
ctx.rotate(cameraBreathe.rotation);
ctx.translate(-center.x, -center.y);

// 2. Shake Layers
if (cameraShake.active) ctx.translate(cameraShake.offsetX, cameraShake.offsetY);
```

## Acceptance Criteria
- [x] Camera slowly drifts in a non-repeating figure-8 pattern.
- [x] Taking damage or using bombs causes the screen to shake violently.
- [x] UI elements (HUD) remain static in screen-space, unaffected by world camera.
