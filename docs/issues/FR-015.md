# [FR-015] High-Frequency Turbulence

## Labels
`MUS`, `enhancement`, `camera`

## User Story
As a player, I want the screen to subtly vibrate on impact so that hits feel devastating but not sickening.

## Proposed Solution

### Overview
A micro-shake system applied to the master canvas translation, used sparingly.

### Implementation Flow
1. Define a global `screenShake` value that decays every frame.
2. Apply `Math.random() * screenShake` to the X/Y canvas translation.
3. Trigger `screenShake = 5` only on specific events (taking damage, bomb exploding).

### Technical Approach
```javascript
if (screenShake > 0) {
  let dx = (Math.random() - 0.5) * screenShake;
  let dy = (Math.random() - 0.5) * screenShake;
  ctx.translate(dx, dy);
  screenShake *= 0.9; // Decay
}
```

### Key Considerations
- NEVER run this continuously during the Surge.
- Keep the amplitude small (1-5 pixels max).

## Acceptance Criteria
- [ ] Screen visibly shakes upon player hit or bomb.
- [ ] Shake smoothly decays to zero in less than 0.5 seconds.
