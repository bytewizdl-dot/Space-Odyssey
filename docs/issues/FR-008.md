# [FR-008] Enhanced Visuals & Shaders

## Labels
`Future`, `graphics`, `rendering`

## User Story
As a player, I want advanced lighting and bloom effects so that the game looks visually stunning and modern.

## Proposed Solution

### Overview
Instead of using expensive full-screen blur filters (which would kill the FPS), we will achieve a "High-End Bloom" look using lightweight **Glow Sprites**. By drawing soft, radial-gradient circles behind projectiles and explosions using the `lighter` composite operation, we make the game "glow" without any heavy pixel-math.

### Implementation Flow
1. Create a `glowTexture` (a small, pre-rendered radial gradient) at startup.
2. Update the `draw()` method of lasers and particles to draw this glow texture *before* drawing the entity itself.
3. Use `ctx.globalCompositeOperation = 'lighter'` only while drawing glows to ensure colors "pop" and brighten as they overlap.

### Technical Approach
```javascript
// Lightweight Bloom: Use 'lighter' composite mode with radial gradients
ctx.globalCompositeOperation = 'lighter';
ctx.drawImage(glowSprite, x, y);
ctx.globalCompositeOperation = 'source-over';
```

### Key Considerations
- **No Blur Filters**: `ctx.filter = 'blur()'` is extremely expensive. We will use pre-rendered soft sprites instead.
- **Additive Blending**: Use `lighter` or `screen` mode to make colors "pop" naturally when they overlap.
- **Selective Rendering**: Only apply glow to high-intensity objects (lasers, core of explosions).

## Acceptance Criteria
- [ ] Lasers and explosions have a soft, glowing halo.
- [ ] The effect can be toggled on/off in the Settings menu for lower-end devices.
- [ ] Base performance does not drop significantly when enabled.
