# [FR-013] Spring-Damped Camera Lag

## Labels
`MUS`, `enhancement`, `camera`

## User Story
As a player, I want the camera to lag behind slightly during sudden acceleration so that I can physically feel the G-force and momentum of the ship.

## Proposed Solution

### Overview
Implement a smooth lerp on the camera's Y offset when the scroll speed changes abruptly.

### Implementation Flow
1. Define a `targetCameraY` based on current speed.
2. Lerp the actual `cameraY` towards `targetCameraY` every frame.
3. Apply `cameraY` to the `ctx.translate()` call.

### Technical Approach
```javascript
// Target sinks lower when speed is high
let targetOffset = (currentSpeed - baseSpeed) * 10;
cameraOffsetY += (targetOffset - cameraOffsetY) * 0.05; // Spring damper
ctx.translate(0, -cameraOffsetY);
```

### Key Considerations
- Must not disconnect the player's visual hitboxes from the actual physical hitboxes.
- Ensure the translation is restored properly.

## Acceptance Criteria
- [ ] Ship visually sinks lower during Surge acceleration.
- [ ] Ship smoothly returns to normal position when Surge ends.
